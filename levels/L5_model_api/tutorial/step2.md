## 这一步你在学什么
第一步你已经把两种 provider 的基础调用修通了。

这一步继续在同一份代码上做扩展，
重点不是自己另写新脚本，
而是看懂现有代码后，把剩下几个能力补齐：
- 多轮对话
- 图像输入
- 流式返回

## 核心概念
### 1) 多轮对话怎么实现
多轮不是接口自动记忆，
而是你把历史消息继续放进 `messages` 里。

所以这一步重点看：
- 当前代码里的 `messages` 够不够多
- 是否真的形成了上下文链

### 2) 图像输入怎么放
这一步不要求你处理真实图片文件，
但要把图像消息结构放对。

常见方式是：
- OpenAI 风格：`image_url` / `input_image`
- Anthropic 风格：`type: image`

### 3) 流式返回怎么处理
流式调用不是一次拿完整 JSON。
你会收到一段一段的数据，
所以代码不能再按“整包 JSON 一次解析”的思路写。

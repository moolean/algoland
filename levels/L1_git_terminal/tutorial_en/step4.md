# Step 4/4 - Lesson: use a branch and make small commits

## Task summary
- Create and switch to branch `feat/onboarding-l1`
- Make at least 2 commits
- One commit message must include `L1 complete`

## What you are learning
Three basic Git habits:
1. create a branch for new work
2. do not squeeze everything into one commit
3. write commit messages that other people can understand

## Core concepts
### 1) Why use a branch
A branch gives your current task its own workspace in Git.

### 2) Why at least 2 commits
This stage includes different kinds of work.
Splitting them into smaller commits makes review and rollback easier.

### 3) How to verify yourself
Use Git to check:
- the current branch name
- recent commits
- whether the required message is present

## Your task

- create branch `feat/onboarding-l1`
- make at least 2 commits
- make sure one commit message includes `L1 complete`

## Self-check

- current branch name
- recent commit history
- whether at least one message includes `L1 complete`

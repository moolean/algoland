# Step 3/4 - 教学：复制配置、修改参数、自己复查

## 任务速览
- 复制 `lab/configs/model_bad.yaml` 到 `out/model_fixed.yaml`
- 把 `lr` 改成 `0.001`，`batch_size` 改成 `32`

## 这一步你在学什么
工程里一个很重要的习惯是：
**先复制原始配置，再在副本上改。**

这样做的好处：
- 不会破坏原材料
- 容易比较前后差异
- 更容易回退

## 本步任务
复制并修复配置：
- 源文件：`lab/configs/model_bad.yaml`
- 目标文件：`out/model_fixed.yaml`

要求：
- `lr: 0.001`
- `batch_size: 32`

## 你会用到的命令
### 1) `cp`
复制文件。

例如：
```bash
cp a.txt b.txt
```
意思是把 a.txt 复制成 b.txt。

### 2) `vim`
打开副本并修改内容。

### 3) `cat`
改完之后再看一遍。

## 为什么这一步很重要
新手常见问题不是“不会改”，而是：
- 改错文件
- 改对了但没保存
- 改完没检查
- 数字写错一位自己没发现

所以这一步最重要的不是手速，而是：
**改完一定自己验证。**

## 推荐做法
1. 先 `cp`
2. 用 `vim` 改副本
3. 用 `cat out/model_fixed.yaml` 检查

## 自检
确认下面两个值都精确存在：
- `lr: 0.001`
- `batch_size: 32`

完成后执行：
```bash
./quest submit
```

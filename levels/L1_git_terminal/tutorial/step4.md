# Step 4/4 - 教学：Git 为什么要分支、为什么要小步提交

## 任务速览
- 创建并切到分支 `feat/onboarding-l1`
- 至少完成 2 次提交
- 其中 1 条提交信息包含 `L1 complete`

## 这一步你在学什么
Git 最核心的三个习惯：
1. 新任务先开分支
2. 不要把所有改动揉成一个 commit
3. 提交信息要能让别人看懂你做了什么

## 本步任务
在仓库内完成：
1. 切换到新分支 `feat/onboarding-l1`
2. 至少 2 次提交
3. 其中一次提交信息包含 `L1 complete`

## 你会用到的命令
### 1) `git checkout -b <branch>`
创建并切换分支。

### 2) `git add <file>`
把你准备提交的文件放进暂存区。

### 3) `git commit -m "..."`
生成一次提交。

### 4) `git log --oneline -n 5`
快速查看最近 5 条提交记录。

### 5) `git branch --show-current`
查看当前所在分支。

## 为什么要至少 2 次提交
因为你这关其实做了不止一种事情：
- 数据提取结果
- 配置修复

把它们拆成两次提交，会比一股脑塞一起更清楚。
这也是以后 code review 更舒服的做法。

## 自检
你应该自己确认：
- 当前分支名是否正确
- 最近提交里是否至少有 2 条和本关有关
- 是否有一条 message 包含 `L1 complete`

完成后执行：
```bash
./quest submit
```

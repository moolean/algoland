# Step 2/4 - 教学：日志统计的经典套路：提取 → 计数 → 排序

## 这一步你在学什么
这一题是终端数据处理里最经典的一种模式：

1. 先提取你关心的字段
2. 把相同内容排到一起
3. 统计每种内容出现次数
4. 再按次数排序
5. 取前几名

以后你做这些事时都会复用它：
- 错误类型统计
- 请求状态码统计
- 用户行为分布统计
- 标签频次统计

## 你会用到的命令
### 1) `awk`
先把“错误类型”提取出来。

### 2) `sort`
让相同的值排到一起。
为什么要先 sort？
因为下面的 `uniq -c` 只会统计“连续重复”的内容。

### 3) `uniq -c`
统计每个值出现了多少次。

例如：
```bash
printf "A\nA\nB\n" | sort | uniq -c
```
输出类似：
```text
      2 A
      1 B
```

### 4) 再次 `sort`
这次是按“次数”排序，而不是按字母排序。

### 5) `head -n 2`
只保留前两名。

## 推荐你的拆分练习
建议你一步一步试，不要一口气写长管道：

先试提取字段：
```bash
awk '...' lab/train.err.log
```

再把它接到：
```bash
... | sort | uniq -c
```

最后再加：
```bash
... | sort -nr | head -n 2
```

如果中间每一步你都看结果，就不会迷糊。

## 任务速览
- 从 `lab/experiments.log` 找出 `val_loss` 最小的原始整行
- 写入 `out/best_run.txt`

## 这一步你在学什么
很多新同学一听到 awk 就紧张，原因通常不是 awk 难，而是：
**还没看懂数据结构就急着写命令。**

这一题真正想教你的不是“背 awk 语法”，而是：
1. 先观察文本结构
2. 找到你关心的字段在哪一列
3. 想办法把字段里的数值拿出来
4. 再做比较

## 本步任务
从：
- `lab/experiments.log`

找出 `val_loss` 最小的整行，写入：
- `out/best_run.txt`

要求：
- 输出的是**原始整行**
- 不是只输出数值

## 第一步：先观察数据
先运行：
```bash
cat lab/experiments.log
```

你会看到类似：
```text
run_id=exp-001 epoch=10 val_loss=0.531
```

现在你要自己观察：
- 第 1 列是什么？
- 第 2 列是什么？
- 第 3 列是什么？

如果你看出来第 3 列是 `val_loss=...`，你就已经完成一半了。

## 你会用到的命令
### 1) `awk`
它适合做“按列处理文本”。

在 awk 里：
- `$1` 表示第 1 列
- `$2` 表示第 2 列
- `$3` 表示第 3 列

例如：
```bash
echo "a b c" | awk '{print $2}'
```
输出：
```text
b
```

### 2) `split(字符串, 数组, 分隔符)`
如果你拿到的是：
```text
val_loss=0.214
```
你可以按 `=` 拆开。

例如思路：
- 左边变成 `val_loss`
- 右边变成 `0.214`

### 3) 变量比较
在 awk 中你可以保存“当前最小值”和“对应那一整行”。

## 建议你先做的小实验
不要急着做最终答案，先试这些：
```bash
awk '{print $3}' lab/experiments.log
```
看清楚第三列长什么样。

再试：
```bash
awk '{split($3,a,"="); print a[2]}' lab/experiments.log
```
看能不能把数字单独打印出来。

如果这一步你看懂了，后面只是把“打印所有值”升级成“保留最小值对应行”。

## 这一步的重点认知
awk 不用一下子全会。
你现在只要掌握三点：
- `$3`：取第三列
- `split()`：拆字符串
- `if`：比较大小

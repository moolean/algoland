# Step 3/3 - 教学：用别名完成一次模拟 SSH 登录

## 任务速览
- 用 `train-node` 完成一次模拟登录
- 把登录结果写入 `out/login_result.txt`

## 这一步你在学什么
前两步你已经：
- 生成了密钥
- 写好了 SSH config

这一步要做的是把配置真正“用起来”。

真实使用时，你不会每次都重新读一遍 config，
而是直接输入：
```bash
ssh train-node
```

所以这一题练的是：
**当别名已经配好后，怎么像真正登录一样去使用它。**

## 核心概念
### 1) 为什么可以直接 `ssh train-node`
因为前一步的 config 已经告诉 SSH：
当你输入 `train-node` 时，应该自动带上：
- 主机地址
- 用户名
- 端口
- 私钥路径

所以这一步你不再关心“字段怎么写”，
而是关心“怎么真正调用这个别名”。

### 2) 为什么这题用“模拟登录”
这里没有真的起一台 SSH 服务器，
所以这一步不做真实网络连接。

但你仍然会完成一次很接近真实使用的动作：
- 带着你的 config
- 带着别名 `train-node`
- 执行一次登录检查

只要它返回 `login ok`，就说明你的配置已经能支持一次正确的别名登录。

## 本步任务
请你用已有的 SSH 配置，完成一次模拟登录，并把输出写入：
- `out/login_result.txt`

题目已经给了你一个模拟登录脚本：
- `lab/mock_ssh_login.sh`

它会读取你的 config，检查：
- 别名是否正确
- 主机地址是否正确
- 用户是否正确
- 端口是否正确
- 私钥路径是否正确

## 自检
提交前先自己检查：
```bash
cat out/login_result.txt
```

如果配置正确，结果里应该能看到：
- `login ok`
- `alias=train-node`

完成后执行：
```bash
./quest submit
```

# Step 1/2 - 教学：命令输出、管道与重定向

## 先安抚一下
如果你看到一条终端命令里同时出现程序名、`|`、`>`，很容易觉得它像黑魔法。
其实不是。

这一步你只需要理解 4 个部件：
- 一个程序，负责**产生信息**
- 一个命令，负责**筛选信息**
- 一个符号，负责**把信息传给下一个环节**
- 一个符号，负责**把最终结果写入文件**

你现在不是在背 shell 语法大全。
你只是在学：**一条命令链，怎样把多个小动作串起来。**

## 这一步你在学什么
今天你要建立一个很重要的终端心智：

> 终端里的长命令，通常不是一个大魔法，
> 而是多个小部件分工合作。

这一步里，这些部件分别扮演：
- **信息源**：先产生内容
- **筛选器**：只保留你关心的部分
- **传送带**：把上一段输出交给下一段
- **保存器**：把最后结果落到文件里

后面几乎每一关都会反复用到这个思路。

## 核心概念
### 1) `vim` 是什么
Vim 是一个文本编辑器。
平时你会用它打开和编辑文件。

但当你写：
```bash
vim --version
```
你并不是要进入编辑界面，
而是在让 Vim 这个程序把自己的版本信息打印出来。

所以在这道题里，`vim --version` 的角色不是“编辑器操作”，而是：
**一个负责产出文本的程序。**

### 2) `|` 是什么
`|` 叫做**管道（pipe）**。
它不是普通命令，而是 shell 里的连接符号。

它的作用是：
**把左边命令产生的输出，交给右边命令继续处理。**

你可以把它想成流水线里的传送带。
左边产出材料，右边接着加工。

例如：
```bash
printf "a\nb\nc\n" | head -n 2
```
这里发生的事是：
- `printf` 先产生 3 行文本
- `|` 把这 3 行交给右边
- `head -n 2` 再只保留前 2 行

### 3) `head -n 1` 是什么
`head` 是一个用来查看“开头几行”的命令。

```bash
head -n 1
```
意思是：
- `-n 1`：只取 1 行

所以在这道题里，它扮演的是：
**筛选器**。

如果左边程序输出了很多行，而你只想保留第一行，`head -n 1` 就很适合。

### 4) `>` 是什么
`>` 叫**重定向**。
它也不是普通命令，而是 shell 的语法符号。

它的作用是：
**把本来要打印到屏幕上的结果，改为写进文件。**

例如：
```bash
echo "hello" > /tmp/demo.txt
```
这时 `hello` 不会显示在终端上，而会被写进 `/tmp/demo.txt`。

⚠️ 你要知道两点：
- `>` 会覆盖目标文件原有内容
- 如果以后想“追加到末尾”，会用 `>>`

## 最小例子
你可以先单独理解每个部件： 

### 先看“程序产生输出”
```bash
vim --version
```
你会看到很多行文本。
这说明：Vim 这个程序确实能把版本信息输出出来。

### 再看“筛选第一行”
```bash
printf "a\nb\nc\n" | head -n 1
```
输出：
```text
a
```

### 再看“写入文件”
```bash
echo "demo" > /tmp/demo.txt
cat /tmp/demo.txt
```
如果你看到 `demo`，说明重定向确实把结果写进了文件。
